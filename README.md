# CNN Parking Lot Classifier

A binary image classification project that identifies whether a parking space is free or occupied using a Convolutional Neural Network (CNN) built with TensorFlow and Keras.

The complete workflow is implemented in `CNN_Notebook.ipynb`, including dataset loading, train/validation/test splitting, data augmentation, hyperparameter tuning, final training, and test evaluation.

## Results

The final notebook was executed with 250 labeled images and produced the following test results:

| Metric | Value |
| --- | ---: |
| Test accuracy | 97.37% |
| Test loss | 0.0365 |
| Macro F1-score | 0.97 |
| Weighted F1-score | 0.97 |

Per-class results:

| Class | Precision | Recall | F1-score | Test images |
| --- | ---: | ---: | ---: | ---: |
| Free (`0_Parqueo`) | 0.95 | 1.00 | 0.97 | 18 |
| Occupied (`1_Parqueo`) | 1.00 | 0.95 | 0.97 | 20 |

## Dataset

The dataset contains 250 images organized into two classes:

| Folder | Label | Meaning | Images |
| --- | ---: | --- | ---: |
| `dataset/0_Parqueo/` | 0 | Free parking space | 120 |
| `dataset/1_Parqueo/` | 1 | Occupied parking space | 130 |

The notebook uses a fixed random seed (`42`) and divides the images as follows:

| Split | Percentage | Images |
| --- | ---: | ---: |
| Training | 70% | 175 |
| Validation | 14.8% | 37 |
| Test | 15.2% | 38 |

The slight percentage difference is caused by rounding whole image counts. The shuffled order is fixed so the split is reproducible.

Images are resized to `128 x 128` pixels and loaded in batches of 16. Labels are inferred directly from the class folder names.

## Data Augmentation

Data augmentation is included inside the CNN and is active only during training:

- Random horizontal flip
- Random rotation up to 5%
- Random zoom up to 20%
- Random contrast adjustment up to 20%

Pixel values are normalized from `[0, 255]` to `[0, 1]` inside the model.

## Model Architecture

The CNN is created by the `build_model(hp)` function and contains:

1. Input layer with shape `(128, 128, 3)`
2. Data augmentation
3. Rescaling layer
4. `Conv2D(32)` and max pooling
5. `Conv2D(64)` and max pooling
6. `Conv2D(128)`
7. Flatten layer
8. Tunable Dense layers
9. Sigmoid output for binary classification

The model uses binary cross-entropy loss, the Adam optimizer, and accuracy as its main metric.

## Hyperparameter Tuning

Keras Tuner performs a Random Search with a maximum of 10 trials. Validation accuracy is used to select the best configuration.

The search space includes:

- Learning rate: `0.01`, `0.001`, or `0.0001`
- Number of Dense layers: 1 to 3
- Units per Dense layer: 32, 64, 128, or 256

The best configuration found in the final run was:

| Hyperparameter | Best value |
| --- | ---: |
| Learning rate | 0.0001 |
| Dense layers | 3 |
| First Dense layer | 32 units |
| Second Dense layer | 32 units |
| Third Dense layer | 128 units |

The Random Search reached a best validation accuracy of 100%. Early stopping is used during both hyperparameter search and final model training. In the final run, training stopped after 16 epochs and restored the weights associated with the lowest validation loss before evaluating the test set.

The notebook reads images directly from the two class folders. The CSV generated by `CreateDataset.py` is an additional dataset index and is not required by the notebook loader.

## Installation

Python 3.12 is recommended.

Create and activate a virtual environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Install the required libraries:

```bash
python -m pip install tensorflow keras-tuner tensorboard numpy matplotlib scikit-learn jupyter
```

The final notebook was run with:

- TensorFlow 2.21.0
- Keras Tuner 1.4.8
- TensorBoard 2.21.0

## Usage

1. Place free-space images in `dataset/0_Parqueo/`.
2. Place occupied-space images in `dataset/1_Parqueo/`.
3. Optionally regenerate the CSV index:

```bash
python CreateDataset.py
```

4. Start Jupyter:

```bash
jupyter notebook CNN_Notebook.ipynb
```

5. Run every notebook cell in order.

The notebook will load and split the images, run the Random Search, train the best model, plot the training history, evaluate the untouched test set, display a confusion matrix, and print a classification report.

## Reproducibility Notes

- Dataset shuffling and Keras Tuner use seed `42`.
- Training data is reshuffled between epochs.
- Validation and test data remain in a stable order.
- The test set is reserved for final evaluation and is not used by Random Search.
- `dataset/`, tuner results, model artifacts, and checkpoints are ignored by Git because they can be large or generated locally.
